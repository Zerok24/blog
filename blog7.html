React: Function vs Class components

Encounrage By the professor, Chris Johnson, to learn something new about React, this week I learn something about the react syntax.

Apart from all the cool stuff that React unlocks for those who learn this framework, like react native, hooks, nextjs, etc, React also gives flexibility in the syntax. You have 2 options to choose from, Class components vs Function components

Class component would look something like this: 

        import React from "react";

            class ClassComponent extends React.Component {
                render() {
                    return <h1>Hello, world</div>;
                }
        }

A function component would look something like this: 

        import React from "react";

        function FunctionalComponent() {
            return <h1>Hello, world</div>;
        }

So basically is just 2 different way to write the same think, but function components are being prefered more and more because is easier to read and it reduces code lines.
